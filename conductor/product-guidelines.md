# Product Guidelines: Manga Fetcher CLI

## Prose Style
- **Technical/Precise:** Feedback and documentation will be direct, concise, and focused on technical accuracy.

## Branding and Tone
- **Minimalist/Utility-First:** Branding will prioritize functionality and performance over aesthetic flair.

## UX Principles
- **Efficiency First:** The CLI will prioritize minimizing user input and automating repetitive download and conversion tasks.
- **Transparency/Feedback:** Users will receive clear progress indicators during downloads and informative error messages in case of failures.
- **Discoverability/Simplicity:** The tool will be easy to learn through standard help commands and sensible defaults for most options.

## Interaction Design
- **CLI Commands:** Use standard flags and subcommands for intuitive navigation.
- **Error Handling:** Fail gracefully with actionable error messages that guide the user on how to resolve the issue.
- **Progress Monitoring:** Provide real-time updates for long-running download and conversion processes.
